# FrameForge 项目交接记录

## 当前目标

基于现有网站继续维护“短剧海报/封面生成导演工具”。网站面向短剧内容生产场景，用户上传剧本 Word 文档和人物参考图后，自动解析剧本、匹配角色、生成导演提示词，并导出可上传到 ChatGPT / GPT Image 2.0 的资源包。

## 重要产品规则

1. 网站不再内置固定 OpenAI API Key。

2. 用户应手动填写自己的 API Key，或使用“导出资源包 + 上传到 ChatGPT”的方式。

3. 解析剧本时，如果剧本文本中没有剧名，则使用上传文件名作为剧名。

4. 角色图优先按文件名自动匹配角色。用户会把图片文件名改成角色名，因此匹配逻辑要优先尊重文件名。

5. 出图时需要同时支持两个比例：

   - 9:16
   - 3:4

6. 每个创意构图版本都要输出 9:16 和 3:4。若用户选择 2 个创意版本，则应产生 4 张图：版本 A 的 9:16、版本 A 的 3:4、版本 B 的 9:16、版本 B 的 3:4。

7. 不同创意版本之间构图必须不同，不能只是同一构图改比例。

8. 视觉语言支持两类：

   - 风格库选择。
   - 根据剧本自动原创视觉语言。

9. 风格库同类海报应合并展示，静止时可重叠，点击后展开，避免冗余卡片。

10. 3:4 海报需要遵守用户提供的安全区规范图，剧名和关键信息不得超出安全区。

## 提示词核心方向

用户对出图质量要求很高。提示词需要强调真人实拍质感，而不是游戏 CG 或动画感。

必须强调：

- 极致真人写实。
- 电影级光影。
- IMAX 胶片摄影机质感。
- Panavision C 系列镜头质感。
- 变形宽银幕电影质感。
- 真实皮肤纹理。
- 自然环境阴影。
- 人物与场景光源一致。
- 人物不要像后期贴上去。
- 剧名必须留在安全区内。

必须规避：

- 游戏 CG 感。
- 动画感。
- 油腻 AI 感。
- 廉价影楼感。
- 人物漂浮。
- 人物像 P 上去。
- 剧名越出安全区。
- 模板化身份乱写。

## 需要特别避免的历史问题

1. 不要把所有剧本都识别成“财阀千金”“复仇继承人”“霸总”等模板身份。

2. 不要在提示词里直接写“禁止改写为财阀千金、复仇继承人”等后端规则型文字。这个应由解析逻辑控制，而不是裸写给 GPT。

3. 不要出现“EP 集数徽标如需出现，仅置于纵向 17.7%-22.5%”这类旧约束。

4. 不要沿用示例剧本或初始剧本里的“阶段三：背叛与惩戒”等内容。

5. “视觉语言与参考范例学习”“构图执行”应在网站解析阶段根据用户上传剧本生成明确内容，不要留给 GPT 再自行识别。

6. 导演提示词里的角色名称必须是真实角色人名或文件名匹配到的角色名，不能出现“图1 时长”“图2 客厅人物”等错误名称。

7. 角色卡底部显示名称、下拉选择结果、导演提示词里的名称必须同步。

8. 选择“自动匹配（按文件名）”后，底部文件名/角色名显示也必须同步更新。

## 资源包应包含

- 剧本解析结果。
- 海报导演提示词。
- 9:16 生成要求。
- 3:4 生成要求。
- 人物参考图片。
- 人物与文件名绑定关系。
- 风格参考信息。
- 9:16 和 3:4 安全区规范。
- 负面约束。
- 上下文隔离提示词。

上下文隔离提示词的目的：用户可能会在已有 ChatGPT 历史记录的对话中制作海报，资源包必须提醒 GPT 只依据当前资源包，不要使用历史记忆、旧剧本或旧角色设定。

## 下一步建议

1. 将网站部署到公司内网服务器或稳定云平台，避免依赖个人电脑开关机。

2. 如果继续使用这台 Mac，建议让 IT 分配固定内网 IP 或内部域名。

3. 继续测试资源包导出，重点看大文件、多人物、多风格图情况下是否稳定。

4. 继续测试人物匹配，重点看中文名、英文名、带空格文件名、多角色同名/近似名。

5. 继续优化安全区约束，因为 GPT Image 对文字排版不能 100% 保证。
