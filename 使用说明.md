# FrameForge 短剧海报导演网站使用说明

## 1. 网站用途

FrameForge 是一个短剧海报/封面生成辅助网站。它的核心作用不是直接替代 ChatGPT 或 GPT Image 2.0，而是把剧本、人物参考图、风格参考、安全区要求整理成一套可执行的海报生成资源包和导演提示词。

用户只需要上传剧本和人物形象图，网站会完成以下工作：

- 解析剧本信息，包括剧名、题材、人物关系、情绪冲突、关键画面。
- 根据上传的人物图片文件名自动匹配角色。
- 生成 9:16 和 3:4 两种比例的海报提示词。
- 按每个创意版本输出两种比例。如果设置 2 个创意版本，则应得到 4 张图：2 个不同构图版本，每个构图各有 9:16 和 3:4。
- 导出资源包，方便用户上传到 ChatGPT / GPT Image 2.0 中继续生图。

## 2. 访问方式

### 临时公网链接

当前可用的临时公网链接：

https://losses-entities-requested-completed.trycloudflare.com

注意：这是临时隧道链接。电脑重启、休眠、网络变化或隧道进程关闭后，链接可能失效，需要重新生成。

### 公司内网访问

如果同事和这台电脑处于同一个公司网络或 VPN，可以尝试以下内网地址：

- http://172.16.199.248:3001
- http://172.16.40.27:3001

注意：内网访问依赖这台电脑保持开机、服务保持运行，并且公司防火墙允许访问 3001 端口。

### 稳定公网部署

如果需要长期稳定的网址，建议部署到 Render、Vercel、公司服务器或公司内网服务器。临时链接不适合长期给多人使用。

## 3. 使用前准备

### 剧本文件

支持上传 Word 文档。建议文件内容包含：

- 剧名
- 剧情简介
- 人物表
- 分集剧情或关键桥段
- 主要人物关系
- 情绪冲突和视觉氛围

如果剧本内容里没有明确写出剧名，网站会优先使用上传文件的文件名作为剧名。

### 人物形象图

人物图片建议按照角色名命名，例如：

- 男主.png
- 女主.jpg
- 沈知意.png
- 陆承安.jpg

网站会根据图片文件名和剧本中的角色名称自动匹配。为了提高匹配准确率，建议文件名和剧本中的角色名完全一致，不要使用“图1”“人物2”“客厅人物”“时长”等无关命名。

### 风格参考

可以选择网站内置的视觉语言，也可以选择“剧本自动原创”，让网站根据剧本题材、人物关系和情绪张力生成原创视觉方案。

同类风格会以合并卡片展示，静止状态下可以重叠预览，点击后展开查看。

## 4. 标准使用流程

1. 打开网站。

2. 上传剧本文件。

3. 上传人物形象图。

   上传前请把图片文件名改成对应角色名。

4. 选择视觉语言。

   可以选择风格库，也可以选择根据剧本自动原创。

5. 点击解析或生成。

   网站会根据剧本内容生成剧名、标签、角色信息、人物绑定和导演提示词。

6. 检查角色匹配。

   如果角色绑定错误，可以手动调整。优先确认图片文件名是否和剧本角色名一致。

7. 检查导演提示词。

   重点检查以下内容：

   - 人物名称是否正确。
   - 人物身份是否来自真实剧本，而不是模板化身份。
   - 构图是否符合当前剧本。
   - 是否同时包含 9:16 和 3:4 版本。
   - 剧名是否被要求放在安全区内。

8. 导出资源包。

9. 将资源包上传到 ChatGPT / GPT Image 2.0，要求其严格按照资源包生成海报。

## 5. 资源包包含什么

导出的资源包通常包含：

- 剧本解析结果。
- 海报导演提示词。
- 9:16 海报生成要求。
- 3:4 海报生成要求。
- 人物参考图片。
- 人物与文件名的绑定关系。
- 风格参考信息。
- 安全区规范图。
- 负面约束。
- 上下文隔离提示词。

上下文隔离提示词的作用是提醒 GPT：本次海报制作只依据当前资源包，不要被之前对话记忆、历史角色设定或旧剧本干扰。

## 6. 在 ChatGPT / GPT Image 2.0 中使用资源包

推荐操作方式：

1. 打开一个新的 ChatGPT 对话。

2. 上传网站导出的资源包。

3. 输入类似指令：

   请严格读取这个资源包，并使用 GPT Image 2.0 生成短剧海报。只依据资源包中的剧本解析、人物参考图、风格规则和安全区规范，不要使用之前对话里的任何记忆或旧设定。

4. 要求输出：

   - 每个创意构图版本输出 9:16 和 3:4 两张图。
   - 如果需要 2 个创意版本，则输出 4 张图。
   - 9:16 和 3:4 是同一创意版本的适配比例，但不同创意版本之间构图必须不同。

5. 生成后重点检查：

   - 人物是否和参考图一致。
   - 人物是否自然融入场景。
   - 是否有真实阴影和光影变化。
   - 是否避免了人物像后期贴上去的感觉。
   - 剧名是否完全位于安全区内。
   - 是否避免游戏 CG 感、动画感和廉价影楼感。

## 7. 画质与风格要求

网站提示词会尽量强调真人实拍感和电影级质感，例如：

- 极致真人写实。
- 电影级光影。
- IMAX 胶片摄影机质感。
- Panavision C 系列镜头质感。
- 变形宽银幕电影质感。
- 自然皮肤纹理。
- 真实环境阴影。
- 人物与场景光源一致。

同时会规避：

- 游戏 CG 感。
- 动画感。
- 油腻 AI 感。
- 廉价影楼感。
- 人物漂浮。
- 人物像被 P 上去。
- 剧名超出安全区。

注意：即使提示词已经限制，GPT Image 2.0 也不能保证每一次都 100% 遵守安全区和排版要求。最终成图仍需要人工检查，不合格时应重新生成或调整提示词。

## 8. API 模式说明

网站已经改为用户手动填写 API 的方式，不再内置固定 API Key。

这样做的原因：

- 避免共享密钥泄露。
- 每个用户使用自己的额度。
- 方便用户自行切换 ChatGPT / OpenAI API 工作方式。

如果 API 不可用，可以使用“导出资源包 + 上传到 ChatGPT”的方式继续完成海报制作。

## 9. 常见问题

### 角色匹配错误怎么办？

优先检查图片文件名。文件名应尽量和剧本中的角色名完全一致。

例如剧本中角色叫“林晚”，图片就命名为“林晚.png”，不要命名为“女主.png”或“图2.png”，除非剧本里也明确使用“女主”作为角色名称。

### 文件名正确但仍然匹配不上怎么办？

检查以下情况：

- 剧本里是否真的出现了同样的角色名。
- 文件名里是否有空格、括号、序号或多余符号。
- 上传的图片是否被浏览器改名。
- 是否存在多个图片使用相似角色名。

必要时可以手动选择角色绑定。

### 导出资源包失败怎么办？

可以按顺序排查：

- 刷新网页后重新上传。
- 减少一次上传的图片数量。
- 压缩过大的参考图片。
- 确认浏览器没有拦截下载。
- 确认剧本文件可以正常解析。
- 避免文件名包含过多特殊符号。

### 同事打不开内网地址怎么办？

请检查：

- 同事是否连接同一个公司网络或 VPN。
- 这台电脑是否开机。
- 网站服务是否仍在运行。
- 公司网络是否允许访问 3001 端口。
- macOS 防火墙是否阻止了 Node 或终端的传入连接。
- 尝试两个内网地址。

### 临时公网链接打不开怎么办？

临时公网链接依赖 Cloudflare Tunnel 进程。如果电脑重启、休眠、断网或进程关闭，链接会失效，需要重新启动隧道并获取新链接。

### 为什么 9:16 和 3:4 构图看起来一样？

同一个创意版本下，9:16 和 3:4 可以保持同一视觉概念，但需要分别适配不同画幅。不同创意版本之间必须是不同构图。

例如设置 2 个创意版本时，应得到：

- 版本 A：9:16
- 版本 A：3:4
- 版本 B：9:16
- 版本 B：3:4

版本 A 和版本 B 的人物站位、镜头距离、情绪瞬间或视觉重心应明显不同。

### 为什么剧名还是可能超出安全区？

因为 GPT Image 2.0 对文字排版和安全区的遵守不是绝对稳定。网站会在资源包里加入安全区规范和强约束，但最终仍需要检查成图。如果剧名越界，应重新生成，或要求模型缩小标题并放回安全区。

## 10. 内网部署建议

如果只给公司同事使用，推荐部署到公司内网机器，例如公司服务器、办公室固定电脑、Mac mini 或 NAS。

基本要求：

- 机器长期在线。
- 固定内网 IP 或公司内部域名。
- Node.js 环境可运行。
- 防火墙允许访问指定端口。
- 同事设备和服务器在同一内网或 VPN。

推荐让 IT 配置：

- 固定 IP。
- 内部域名，例如 `poster.company.local`。
- 端口转发或反向代理。
- 自动重启服务。
- HTTPS 内网证书。

### 当前这台电脑的开机自启配置

当前已在这台 Mac 上配置 macOS 登录后自动启动网站服务。

服务名称：

`com.frameforge.poster-studio`

启动配置文件：

`/Users/apple/Library/LaunchAgents/com.frameforge.poster-studio.plist`

项目内备份配置：

`/Users/apple/Documents/短剧海报项目/ops/com.frameforge.poster-studio.plist`

自动启动端口：

`3001`

同事可优先访问：

`http://172.16.199.248:3001`

注意：这个配置可以解决“电脑重启后不用手动重新启动网站服务”的问题，但不能解决“电脑关机时同事仍然访问”的问题。只要这台电脑关机、休眠、断网，网站仍然不可访问。

## 11. 使用注意事项

- 不要把个人 API Key 写死在网站代码里。
- 不要把 API Key 发给他人。
- 上传人物图片前先确认版权和授权。
- 上传剧本前确认内容可以在对应平台处理。
- 资源包里可能包含剧本、人物图和项目资料，请不要随意外传。
- 生成海报后需要人工审核人物一致性、文字安全区、画面质感和平台规范。

## 12. 推荐工作流

日常推荐使用以下流程：

1. 上传剧本。
2. 上传按角色名命名的人物图。
3. 选择“剧本自动原创”或合适风格库。
4. 检查角色匹配。
5. 导出资源包。
6. 上传资源包到 ChatGPT / GPT Image 2.0。
7. 生成每个创意版本的 9:16 和 3:4 海报。
8. 检查安全区和人物一致性。
9. 不合格则基于问题重新生成。
